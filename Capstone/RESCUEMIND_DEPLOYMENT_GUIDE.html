🏗️ Architecture Overview

RescueMind AI uses a split architecture — the frontend runs on Vercel (serverless, fast CDN) and the heavy AI inference runs on Render (persistent Node.js server). This is the standard pattern in 2026 for apps using Transformers.js or other ML models that need a warm process.

🔄 How the proxy works

When a user submits a report on /report:

  1. The Next.js API route /api/classify receives the request
  2. If AI_SERVICE_URL is set, it proxies the request to https://rescuemind-ai-service.onrender.com/api/classify
  3. The Render AI service runs Transformers.js, classifies the report, and returns the result plus the embedding vector for duplicate detection
  4. The Next.js route then performs duplicate check + Gemini enrichment + stores to Supabase
  5. The user sees the classification result immediately

💡 If AI_SERVICE_URL is not set, the app runs entirely on Vercel using Transformers.js inline. This works but has slower cold starts.

🧠 Deep Dive: The AI Service

This is the most important piece to understand before deploying. Let's break down exactly what the AI service does and why it needs its own server.

What is Transformers.js?

Transformers.js is an open-source library that runs Hugging Face transformer models directly in Node.js (no Python needed). RescueMind uses the paraphrase-multilingual-MiniLM-L12-v2 model — a lightweight, multilingual sentence embedding model trained to understand Tagalog and English equally well.

PropertyValue
Model nameXenova/paraphrase-multilingual-MiniLM-L12-v2
TypeSentence embedding (feature extraction)
Languages50+ languages including Tagalog, Cebuano, Ilocano
Download size~85 MB (ONNX quantized)
RAM at inference~300 MB
Inference time~500 ms per sentence (first run slower)
LicenseMIT — fully open source
First-load time~5-10 seconds (download + cache model)

How classification works (step by step)

  1. Embedding: The citizen's complaint text is converted into a 384-dimensional vector (embedding) — a mathematical representation of its meaning
  2. Similarity matching: This embedding is compared against 10 pre-computed category embeddings (Flood, Road Damage, Garbage, etc.) using cosine similarity
  3. Threshold check: If the best match is below 60% confidence, it's flagged as "Needs Human Review"
  4. Duplicate detection: The same embedding is also compared against the last 20 stored complaints. If similarity > 85%, the citizen sees a yellow warning: "Mayroon nang 2 kahalintulad na ulat mula sa inyong lugar."
  5. Optional enrichment: If GEMINI_API_KEY is set, the report text + classification is sent to Gemini 2.0 Flash to generate an empathetic Tagalog explanation
⚡ Why serverless (Vercel) struggles with ML

Serverless cold starts: Every time a Vercel function wakes up after being idle, it must re-download or re-load the 85 MB model into memory. This takes 5–10 seconds before the first classification can run. For a citizen waiting for their report to be processed, this is a terrible experience.

Memory limits: Vercel's free tier gives 1 GB of memory, but the model (~300 MB at inference) combined with Next.js itself (~200 MB) can cause memory pressure during peak usage.

Render's solution: A persistent Node.js server loads the model once at startup and keeps it in memory. Every subsequent classification completes in ~500ms — no cold starts, no memory spikes.

1 Create a Supabase Project

Supabase provides the database and authentication for RescueMind AI. It's free for small projects.

  1. Go to supabase.com and click Start your project
  2. Sign in with GitHub (recommended) or email
  3. Click New project
  4. Fill in:
    • Name: rescuemind-ai
    • Database Password: Create a strong password — save this safely!
    • Region: Southeast Asia (Singapore) — fastest for Philippines
    • Pricing Plan: Free
  5. Click Create new project — wait ~2 minutes for provisioning
  6. Once created, go to Project Settings → API in the left sidebar
📋 Copy these values

  • anon public key — a long base64 string starting with eyJ...

9 Connect Frontend → AI Service

Now wire them together. If you set AI_SERVICE_URL in Vercel (Step 8), the frontend is already pointing at your Render AI service. Let's verify:

  1. Go to Vercel Dashboard → rescuemind-ai → Settings → Environment Variables
  2. Confirm AI_SERVICE_URL is set to https://rescuemind-ai-service.onrender.com
  3. Go to Deployments → find the latest deployment → click Redeploy
  4. Wait for the build to finish
📡 How the connection works

When AI_SERVICE_URL is set, the Next.js API route /api/classify acts as a proxy:

Citizen submits report → /api/classify (Vercel) → POST https://rescuemind-ai-service.onrender.com/api/classify → Transformers.js classifies (warm, ~500ms) → Returns: category + urgency + office + embedding → Vercel checks duplicates + calls Gemini (if key set) → Returns final result to citizen

💡 If AI_SERVICE_URL is empty, Vercel runs Transformers.js inline (slower cold starts but works).

⚠️ After adding AI_SERVICE_URL, you must Redeploy

Environment variable changes don't take effect until the next deployment. Trigger a manual re-deploy from the Vercel dashboard.

🔧 Troubleshooting

❌ AI Service health check returns "model": "loading"

Cause: The 85 MB model is still downloading from Hugging Face on first deploy. This can take 1-3 minutes.

Fix: Wait 30 seconds and refresh. The model is cached for subsequent deploys. If it never loads, check Render logs for download errors (rare — usually a network issue).

❌ "Module not found: @xenova/transformers" on Vercel

Cause: Next.js tries to bundle Transformers.js as a client-side module, but it's a Node.js library.

Fix: Make sure your next.config.js has:

const nextConfig = { serverExternalPackages: ["@xenova/transformers"], };

This tells Next.js to keep the package on the server only.

❌ Report submission fails — "AI Service not reachable"

Cause: The AI_SERVICE_URL env var is set but the Render service isn't running or the URL is wrong.

Fix:

  1. Check your Render AI service is live: visit https://rescuemind-ai-service.onrender.com/api/health
  2. Verify AI_SERVICE_URL in Vercel env vars has no trailing slash
  3. Redeploy the Vercel project after changing env vars
  4. If Render is on the Free plan and was idle, the first request takes 5-10s to wake up
❌ Classification always returns "Needs Human Review"

Cause: The confidence threshold isn't being met. This can happen if the Transformers.js model fails to load and falls back to the character-level hash embedding (low quality).

Fix: Check your Render AI service logs. If the model fails to load, the service uses a simple character-counting fallback that produces poor embeddings. Force a re-deploy of the AI service on Render to re-download the model.

❌ Auth not working — "Invalid login credentials"

Fix: Go to Supabase → Authentication → Providers → Email and make sure "Enable email confirmations" is OFF for testing. For production, keep it ON.

❌ Render build fails — "Could not find package.json"

Cause: Root directory is set to / instead of Capstone/ai-service.

Fix: Update the Root Directory setting in Render to Capstone/app/ai-service. If using Blueprint, verify render.yaml has rootDir: app/ai-service and it's at the repo root.

❌ Gemini not responding — shows offline explanation

This is expected if the key is missing or invalid. To fix:

  1. Verify GEMINI_API_KEY is set in Vercel env vars (not Render — Render doesn't call Gemini)
  2. Ensure it's not the placeholder your-gemini-api-key
  3. Check the key has billing enabled (free tier should work)
❌ AI Service runs out of memory on Render Free (512 MB)

Cause: The Transformers.js model uses ~300 MB + Node.js uses ~100 MB = ~400 MB total. On the Free plan's 512 MB limit, this is tight but functional.

Solutions:

  • Stay on Free: works fine for 1-2 concurrent requests
  • Upgrade to Starter ($7/mo, 2.5 GB): handles 10+ concurrent requests
  • Reduce model memory: set { quantized: true } in loadModel() (already configured)
